# Настройка бота для мессенджера MAX

Пошаговая инструкция по созданию и запуску бота «Воздух НСК» в MAX.

---

## 1. Требования

- Юридическое лицо или ИП — резидент РФ (создавать ботов могут только они)
- Регистрация на платформе MAX для партнёров
- Одна организация может создать до 5 ботов

---

## 2. Получение токена

1. Открой мессенджер **MAX** на телефоне или в браузере
2. Найди бота **@MasterBot**
3. Отправь команду `/create`
4. Следуй подсказкам:
   - **Ник бота** — минимум 11 символов, латиница, заканчивается на `bot` или `_bot` (например: `vozduh_nsk_bot`)
   - **Имя бота** — до 16 символов, отображается в чате (например: «Воздух НСК»)
5. После создания получишь **токен** — сохрани его, он показывается один раз

### Полезные команды в @MasterBot

| Команда | Описание |
|---------|----------|
| `/get_token` | Показать текущий токен |
| `/refresh_token` | Обновить токен |
| `/revoke` | Отозвать токен |
| `/set_description` | Описание бота (до 512 символов) |
| `/set_picture` | Загрузить фото профиля |

---

## 3. Настройка проекта

1. Перейди в папку бота:
   ```bash
   cd max-bot
   ```

2. Скопируй пример конфига:
   ```bash
   cp .env.example .env
   ```

3. Открой `.env` и укажи токен:
   ```
   MAX_BOT_TOKEN=твой_токен_из_MasterBot
   ```

4. Установи зависимости (если ещё не установлены):
   ```bash
   npm install
   ```

---

## 4. Запуск

```bash
npm start
```

В консоли появится: `Бот MAX «Воздух НСК» запущен. Ожидаю сообщения...`

---

## 5. Модерация

После создания бот проходит модерацию. Пока она не пройдена, пользователи не смогут его найти. Обычно проверка занимает до нескольких дней.

---

## 6. Деплой (VPS, Railway и т.п.)

При деплое добавь переменную окружения:
- `MAX_BOT_TOKEN` — токен из @MasterBot

Бот использует long polling по умолчанию. Для production можно настроить webhook (см. [документацию MAX](https://dev.max.ru/docs/chatbots/bots-coding/library/js)).
