# Отчёт аудита проекта vozduh-updated-fixed3photo

**Дата:** Март 2025

## Выявленные и исправленные проблемы

### 1. Мёртвый код
- **safeJson** — функция не вызывалась, удалена
- **WORKS и WorksSection** — заменены на WorksMarquee, неиспользуемый код удалён
- **CSS worksMarquee** — анимация в index.css (works-marquee-track, works-marquee-paused) не использовалась (WorksMarquee работает через JS), удалена

### 2. Производительность
- **WorksMarquee** — использование `translate3d()` вместо `translateX()` для GPU-ускорения анимации прокрутки
- **BrandMarquee** — улучшены ключи React: `key={i}` заменён на `key={\`${b}-${i}\`}` для корректного reconciliation при дублировании брендов

### 3. Доступность (a11y)
- **prefers-reduced-motion** — добавлена поддержка пользователей с «Уменьшить движение»:
  - Отключена плавная прокрутка (scroll-behavior: auto)
  - Остановлена CSS-анимация бегущей строки брендов
  - Отключена авто-прокрутка WorksMarquee при соответствующей настройке системы

### 4. Анимации
- Плавность анимаций проверена, easing `[0.22, 1, 0.36, 1]` используется консистентно
- Исправлено использование `translate3d` для GPU-слоёв
- `will-change: transform` применяется только к анимируемым элементам (marquee-track)

## Рекомендации на будущее

1. **Разделение App.jsx** — файл ~2800 строк; целесообразно вынести компоненты (модалки, секции) в отдельные файлы
2. **Вынос данных** — complexSolutions, turkovCategories, ARTICLES и др. перенести в `src/data/`
3. **Вынос API** — postLead, fetchGeminiResponse оформить как `src/api/leads.js`, `src/api/gemini.js`
4. **Мемоизация** — обернуть тяжёлые дочерние компоненты в `React.memo` при появлении узких мест по производительности
